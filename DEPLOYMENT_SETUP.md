## 🔑 Setting Up GitHub Secrets

To enable automatic deployment, you need to add your FTP credentials as **GitHub repository secrets**.

### Steps:

1. **Go to your GitHub repository**
   - Navigate to: `Settings` → `Secrets and variables` → `Actions`

2. **Add three secrets** (click "New repository secret"):

   | Secret Name | Value |
   |---|---|
   | `FTP_SERVER` | `ftp.ideallasers.com` |
   | `FTP_USERNAME` | `u73860686.ideal` |
   | `FTP_PASSWORD` | `[Your FTP password]` |

3. **Example:** After clicking "New repository secret":
   ```
   Name: FTP_SERVER
   Secret: ftp.ideallasers.com
   ```

### ⚠️ Security Notes

- Secrets are **encrypted** and never visible in logs
- Only the workflow has access to these values
- Never commit credentials to the repository
- Regenerate the password if compromised

## 🚀 Automatic Deployment

Once secrets are configured:

1. **Push to main branch**
   ```bash
   git add .
   git commit -m "Deploy to production"
   git push origin main
   ```

2. **GitHub Actions automatically**:
   - ✓ Checks out code
   - ✓ Installs dependencies
   - ✓ Builds static export (`npm run build`)
   - ✓ Uploads `/out/` files to FTP at `/public_html/`

3. **Monitor deployment**:
   - Go to `Actions` tab in your GitHub repository
   - Click the latest workflow run to see logs
   - Status: `✅ Success` or `❌ Failed`

### Workflow File

Location: `.github/workflows/deploy.yml`

- Triggers: Push to `main` branch or manual dispatch
- Deploy path on server: `/public_html/`
- Syncs deletions: `false` (preserves server files not in build)

## 🐛 Troubleshooting

| Issue | Solution |
|---|---|
| FTP upload fails | Verify FTP credentials in GitHub Secrets are correct |
| Workflow doesn't run | Ensure branch is `main` and workflow file is in `.github/workflows/` |
| Site shows old content | Clear browser cache and FTP server cache (may take 5-10 mins) |
| 404 errors on routes | Verify `/out/` folder structure includes all route directories |
